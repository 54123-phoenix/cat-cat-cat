---

# 一、项目概述

## 1.1 项目定位

**猫猫社区**是一个基于 AI 图像识别的校园猫猫数字档案与偶遇记录平台。

面向两类用户：
- 普通同学：拍照识猫、浏览猫档案、记录偶遇
- 管理者（爱猫协会）：维护猫猫档案、审核识别结果、管理数据

---

## 1.2 项目目标

> 用 AI 让每一只校园猫被看见、被记录、被记住。

核心体验闭环：

```text
拍照 → AI识别 → 猫档案 → 偶遇记录 → 社区沉淀
```

---

# 二、系统整体架构

系统采用：

> 小程序前端 + 云端业务服务 + 本地AI推理服务（GPU）

三层混合架构。

```text
┌────────────────────────────┐
│        小程序 / Web前端        │
│ 首页 / 识别 / 档案 / 地图 / 我的 │
└─────────────┬──────────────┘
              │ HTTPS API
              ▼
┌────────────────────────────┐
│        云端 FastAPI 服务       │
│                                │
│ 用户系统 / 猫档案 / 偶遇记录     │
│ 图片管理 / AI请求转发 / 管理后台 │
└───────┬──────────┬─────────┘
        │          │
        ▼          ▼
┌────────────┐  ┌────────────────┐
│  SQLite/MySQL │  │  图片存储系统     │
│  猫档案数据库  │  │ 用户上传/猫照片   │
└────────────┘  └────────────────┘

              AI识别请求
                    │
                    ▼
┌────────────────────────────┐
│     本地 AI 推理服务（GPU）    │
│                                │
│ CLIP 图像编码                  │
│ FAISS 向量检索                 │
│ 猫身份识别                     │

```

# 五、AI识别模块（本地GPU服务）

---

## 5.1 技术栈

- PyTorch
- CLIP（图像特征提取）
- FAISS（向量检索）
- OpenCV / PIL

---

## 5.2 核心流程

### Step1：图像输入

用户上传猫照片。

---

### Step2：特征提取（CLIP）

```text
Image → CLIP Encoder → Feature Vector
```

---

### Step3：向量检索（FAISS）

```text
Feature Vector → FAISS → Top-K 相似猫
```

---

### Step4：结果输出

返回：

```json
{
  "cat_id": 1,
  "name": "小白",
  "confidence": 0.92
}
```

---

## 5.3 识别策略（置信度分级）

| 置信度 | 行为 |
|--------|------|
| > 0.90 | 自动确认 |
| 0.70~0.90 | 候选列表 |
| < 0.70 | 进入人工审核 |

# 六、数据库设计

---

## 6.1 cats（猫档案表）

```sql
CREATE TABLE cats (
    id INTEGER PRIMARY KEY,
    name TEXT,
    color TEXT,
    personality TEXT,
    story TEXT,
    location TEXT,
    avatar TEXT,
    created_at DATETIME
);
```

---

## 6.2 cat_images（猫参考图）

```sql
CREATE TABLE cat_images (
    id INTEGER PRIMARY KEY,
    cat_id INTEGER,
    image_path TEXT,
    embedding_path TEXT,
    created_at DATETIME
);
```

---

## 6.3 sightings（偶遇记录）

```sql
CREATE TABLE sightings (
    id INTEGER PRIMARY KEY,
    cat_id INTEGER,
    image_path TEXT,
    location TEXT,
    confidence REAL,
    created_at DATETIME
);
```
